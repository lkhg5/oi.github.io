08. Collections - Stack and Queue

035ND C01 L01 A09 QUEUE AND STACK

Here we are going to introduce two interesting data structures, stack and queue.

Stack follows last-in-first-out (LIFO) principle. That’s when you add an element, you add that to the top, when you remove an element, you also remove it from the top. This is similar to the two methods that we just learned from LinkedList, offerFirst, and removeFirst, which are equivalent to Stack’s push and pop.

On the other hand, Queue follows first-in-first-out (FIFO) principle. An excellent example of a queue is a line of customers waiting to buy ice cream. The first customer will get the ice cream first, and when they get the ice cream, they will leave the queue. This is similar to the two methods we learned from LinkedList, offerLast, and removeFirst, which are equivalent to Queue’s enqueue, and dequeue.

Below is the solution code with comments that explain how the code works:

``` java
import java.util.Stack;

public class Calculator {

public static int calculate(String s) {
    int len;
    // Check if the string is null or empty (length = 0). If the condition is met, return 0 as there is nothing to calculate
    if (s == null || (len = s.length()) == 0) {
        return 0;
    }
    // Initiate a Stack for storing the numbers represented by the string
    Stack<Integer> stack = new Stack<>();
    // Declare a variable "num" to store each number that exists in the string expression (each number can have one digit (eg, 5) or more (eg, 25))
    int num = 0;
    // Declare a variable "sign" to store the operator, since this calculator works on positive integers only, the initial value of the sign can be set to "+"
    char sign = '+';
    // Iterate through the string to check each character and take the proper action based on whether it's a number or an operator
    for (int i = 0; i < len; i++) {
        if (Character.isDigit(s.charAt(i))) {
        /* If the current character is a digit, we have two possible cases. It's either that this digit is followed by a sign, which means we have a single digit number (eg, 5), 
        or the digit is followed by another digit, meaning that we have a number that consists of more than one digit (eg, 25). To take the second case in consideration, 
        we simply multiply the previous digit (which will be stored in the variable num) with 10, add the current digit to it, then we store the resulting number in the variable num. 
        This will also work properly if we have the first case, where num will be 0 and multiplying 0 by 10 will yield 0, which means we'll just be storing the current single digit number in num variable. */
            num = num * 10 + s.charAt(i) - '0';
        }
        // If the current character is an operator ( +, -, or *) or if it's the last character, we check the sign and do the necessary Stack operation accordingly
        if (!Character.isDigit(s.charAt(i)) || i == len - 1) {
            // If the sign represents subtraction, this means we need to push the number (which was calculated in the previous iteration) with a negative value
            if (sign == '-') {
                stack.push(-num);
            }
            // If the sign represents summation, this means we need to push the number (which was calculated in the previous iteration) as is (because it's already positive)
            // The first push operation will happen through the if statement below, because we set our initial sign value to be "+"
            if (sign == '+') {
                stack.push(num);
            }
             /* If the sign represents multiplication, this means we need to pop the topmost number in the stack, multiply it by the current number (which was calculated in the previous iteration), 
             then push the result to the stack */
            if (sign == '*') {
                stack.push(stack.pop() * num);
            }
            // Since we pushed the proper number to the stack, it's time to store the value of the current operator (+, -, or *) so we can take the proper action in the next iterations
            sign = s.charAt(i);
           // Since we already pushed num to the stack, we need to reset it so we can do the next calculations correctly
            num = 0;
        }
    }
    // Declare a variable ans to store the answer
    int ans = 0;
    // Sum up all the numbers in the stack to get the result
    for (int i : stack) {
        ans += i;
    }
    //Return the result
    return ans;
}

} ```

Here is an example that you can use to trace the code and understand it more:

Suppose we have the expression: "3-2", the following will happen when we iterate through the string:

Iteration 1 (index 0): Only the first if statement will be executed and 3 will be stored in "num". Please note that "sign" will be set to "+" before this iteration.

Iteration 2 (index 1): Only the second if statement will be executed, where 3 will be pushed to the stack and since the current sign is "-", the value of "sign" will be updated to "-".

Iteration 3 (index 2): The first if statement will be executed and 2 will be stored in "num". The second if statement will be executed as well since this is the last index, where the sign will be checked which will be equal to "-", then -2 will be pushed to the stack.

After the loop: We'll sum 3 and -2 and return the result which is 1.
