01. Introduction

Overview

In this lesson, you will deploy your application in a container. For now, consider a container as an isolated computing environment in a given machine. In our case, this container will be running inside a virtual machine on the cloud, called EC2 instance. Next, we will learn to "connect" our code residing in a Github repository to the container with Jenkins's help. This "connection" is called CI/CD pipeline. The diagram below shows the overarching theme of the current lesson.

Lesson Outline

ND035 C04 L05 A01 Definition Of CI-CD V2

In this lesson, you will learn the various technologies that aid with the process of Continuous Integration / Continuous Delivery (CI/CD). By the end of the lesson, you will be able to:

  • Create a Git repository in GitHub, and run a few basic commands

  • Introduce the Docker, Container, and Images

  • Create and connect to the EC2 instances on AWS,

  • Configure the Jenkins console to automate the CI/CD pipeline

The first few pages of this lesson will make you familiar with the necessary third party tools/softwares, such as Github, AWS, Docker, and Jenkins. All these concepts play an essential role in the different phases of CI/CD pipeline. Bear with us for learning these tools from their official documentation, though we will demonstrate the usage in our videos.
