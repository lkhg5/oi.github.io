02. Spring Boot Template Engine - Thymeleaf

035ND C01 L03 A03 SPRINGBOOT WITH THYMELEAF INTRO

Thymeleaf is a template engine that is based on attributes. This simply means that if you have any variables that reside on the server and you want to use them in your page, you just need to refer to them by using some attributes in your HTML code (we'll see how in the coming lessons).

When a browser wants to display a web page, it builds a DOM (Document Object Model) of that page. DOM is a tree representation of your HTML document, where each element (tag) in your HTML code is defined as an object that can have a parent and one or more children. In other words, DOM is an object-based representation of your web page and it is the way browsers represent a page in memory, i.e., a browser creates a DOM from an html code and stores it in memory. A developer can access DOM through some languages like javascript and change things in the page (like some text, image, etc.) without the need of changing the HTML of that page.

Thymeleaf modifies the DOM (which resides in memory) to make it reflect the values of the variables on the server side (those that were referenced in the HTML code), and by that the web page will be dynamic and will have the values of those variables displayed. It should be mentioned that Thymeleaf performs all of its processes in memory (including modifying DOM), that's why it's considered an in-memory template engine.

Let’s take an example about DOM tree, look at the HTML code below: 

<!DOCTYPE html>
<html>
    <head>
           <title>Demo</title>
    </head>
    <body>
           <h1>What is DOM?</h1>
           <p>DOM is an object-based representation of your web page and it is the way browsers represent a page in memory</p>
    </body>
</html>

As you can see, html is the root element and it has two children, head and body. Head has one child which is title which has the value of “Demo”. Body has two children, the first one is h1 and it has the value of “What is DOM?”, the second one is p and it has the value of “DOM is an object-based representation…”. Instructions

  1. Go to https://start.spring.io/ and create a new Spring Boot project named spring-boot-thymeleaf with Thymeleaf and Spring Web being added as dependencies.
  2. Download, unzip, and import the project.
  3. Create a controller directory under com.example.springbootthymeleaf. Then, create a UserController Java class under the controller directory. Your code should look like the one below: 
@Controller
public class UserController {

   @RequestMapping("demo")
   public String demo(Model model) {
       model.addAttribute("message", "Hello Thymeleaf");
       // return to templates/demo.html page.
       return "demo";
   }
}
  1. Create a demo.html file under resources/template directory. Your code should look like the one below:
<!DOCTYPE html>
<html lang="en" xmlns:th="http://www.w3.org/1999/xhtml">
<head>
   <meta charset="UTF-8">
   <title>Thymeleaft Demo</title>
</head>
<body>
   <span th:text="${message}"></span>
</body>
</html>
  1. Run the application, and go to http://localhost:8080/demo, you should see “Hello Thymeleaf” message printed on the web page.

035ND C01 L03 A04 SPRINGBOOT WITH THYMELEAF

That’s our Spring Boot application using Thymeleaf, easy right?

Compared to freemarker, it’s easier to use. One thing you need to add to your html header is xmlns:th="http://www.w3.org/1999/xhtml", which tells the html that we are using Thymeleaf engine.
