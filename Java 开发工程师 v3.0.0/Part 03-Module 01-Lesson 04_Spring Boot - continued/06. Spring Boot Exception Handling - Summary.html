06. Spring Boot Exception Handling - Summary

035ND C01 L04 A08 EXCEPTION SUMMARY

Let’s recap these Spring Boot exception handling ways and how to implement them.

  1. Creating a custom error page -> create an error.html page, when errors happen, users will be redirected to this page

  2. Using @ExceptionHandler -> handle exceptions inside each controller by providing exception handler methods to handle different exceptions

  3. Using @ControllerAdvice + @ExceptionHandler -> create a separate class and write all exceptions handler methods that you want them to handle different exceptions across the entire application

  4. Configuring a SimpleMappingExceptionResolver class -> rather than adding one handler method per exception, create a key-value mapping that maps each exception to a view and let system look for the error

  5. Using a custom HandlerExceptionResolver class -> use reflection to find what type of exception it’s then add the proper view to a ModelAndView object
